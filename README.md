# Лабораторная работа №4. Продвинутые объекты в JavaScript

## Цель работы

Познакомиться с классами и объектами в JavaScript, научиться создавать классы, использовать конструкторы и методы, а также реализовать наследование.

## Условие

Создайте консольное приложение, моделирующее систему инвентаря, где можно добавлять предметы, изменять их свойства и управлять ими.

### Шаг 1. Создание класса `Item`

Создаем класс `Item`, который будет представлять предмет в инвентаре.

- **Поля класса**:
  - `name` – название предмета.
  - `weight` – вес предмета.
  - `rarity` – редкость предмета (`common`, `uncommon`, `rare`, `legendary`).
- **Методы**:
  - `getInfo()` – возвращает строку с информацией о предмете.
  - `setWeight(newWeight)` – изменяет вес предмета.

### Шаг 2. Создание класса `Weapon`

Создаем класс `Weapon`, который расширяет `Item`.

- **Дополнительные поля**:
  - `damage` – урон оружия.
  - `durability` – прочность (от 0 до 100).
- **Методы**:
  - `use()` – уменьшает `durability` на 10 (если `durability > 0`).
  - `repair()` – восстанавливает `durability` до 100.

## Контрольные вопросы

### 1. Какое значение имеет `this` в методах класса?
   
В методах класса this ссылается на конкретный экземпляр объекта, который был создан с помощью оператора new. Через this осуществляется доступ к свойствам и другим методам этого конкретного объекта.

### 2. Как работает модификатор доступа `#` в JavaScript?
   
 "#" делает свойство приватным.
 
Это значит, что к нему можно обращаться только внутри класса. Попытка обратиться к #property снаружи класса вызовет синтаксическую ошибку. Это обеспечивает инкапсуляцию, защищая внутренние данные объекта от прямого изменения.

### 3. В чем разница между `классами` и `функциями-конструкторами`?
   
Функции-конструкторы — это более старый подход к созданию обьектов и организации наследования, основанный на работе с prototype, где разработчик вручную настраивает наследование и методы. Классы — это современный синтаксис, который делает тот же механизм более понятным и удобным: вместо явной работы с прототипами используется ключевое слово extends, а конструктор и методы описываются внутри класса. Таким образом, классы упрощают чтение и написание кода, хотя внутри они всё равно работают через прототипы.
